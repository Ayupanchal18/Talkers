# ✅ Pre-Deployment Checklist

Use this checklist before deploying to ensure everything is configured correctly.

## 📋 Code Preparation

- [ ] All code committed to Git
- [ ] `.env` files are in `.gitignore` (never commit secrets!)
- [ ] `render.yaml` file exists in root directory
- [ ] Both `client/` and `server/` have `package.json` with correct scripts
- [ ] Database migrations are up to date locally

## 🔐 Security Check

- [ ] Never commit `.env` files with real credentials
- [ ] JWT secrets will be auto-generated by Render (or manually set)
- [ ] Database connection strings will be injected by Render
- [ ] No hardcoded API URLs in client code (using env vars)

## 📦 Build Commands Verification

### Server (Backend)
- [ ] Build command: `cd server && npm install && npm run build && npx prisma generate`
- [ ] Start command: `cd server && npx prisma migrate deploy && npm start`
- [ ] Health check endpoint exists: `/health`

### Client (Frontend)
- [ ] Build command: `cd client && npm install && npm run build`
- [ ] Publish directory: `client/dist`
- [ ] React Router rewrite rule configured

## 🗄️ Database

- [ ] PostgreSQL selected (not MySQL or MongoDB)
- [ ] Database name: `vidss`
- [ ] Region matches backend region
- [ ] Prisma schema is up to date

## 🌐 Environment Variables

### Backend Required:
- [ ] `NODE_ENV` = `production`
- [ ] `PORT` = `10000`
- [ ] `DATABASE_URL` = (from Render database)
- [ ] `JWT_SECRET` = (auto-generated or random 64-char string)
- [ ] `JWT_REFRESH_SECRET` = (auto-generated or random 64-char string)
- [ ] `FRONTEND_URL` = (your frontend URL after deployment)

### Frontend Required:
- [ ] `VITE_API_URL` = (your backend URL after deployment)

## 🚀 Deployment Method

Choose one:

### Option A: Blueprint (Automated) ✨ RECOMMENDED
- [ ] Push code to GitHub with `render.yaml`
- [ ] Go to Render Dashboard → New + → Blueprint
- [ ] Select repository
- [ ] Click "Apply"
- [ ] Wait 5-10 minutes

### Option B: Manual Setup
- [ ] Create PostgreSQL database first
- [ ] Deploy backend web service
- [ ] Deploy frontend static site
- [ ] Update environment variables with URLs
- [ ] Verify health endpoint

## ✅ Post-Deployment Verification

- [ ] Backend health check returns 200: `https://your-backend.onrender.com/health`
- [ ] Frontend loads without errors: `https://your-frontend.onrender.com`
- [ ] Can register a new user
- [ ] Can login successfully
- [ ] Can create a meeting room
- [ ] Can join a meeting from another browser/device
- [ ] Video/audio streams work between peers
- [ ] Chat messages send and receive
- [ ] Socket.IO connection is stable (check browser console)

## 🐛 Common Issues

### Backend won't start
- Check logs in Render Dashboard
- Verify `DATABASE_URL` is correct
- Ensure Prisma migrations ran successfully

### Frontend shows blank page
- Check browser console for errors
- Verify `VITE_API_URL` points to backend
- Check CORS configuration in backend

### Socket.IO connection fails
- Check `FRONTEND_URL` in backend env vars
- Verify WebSocket support (should work on Render)
- Try after 30 seconds if backend was sleeping

### Video/audio not working
- HTTPS is required (Render provides this ✅)
- Check browser permissions for camera/microphone
- Try in Chrome/Edge (best WebRTC support)
- Check browser console for WebRTC errors
